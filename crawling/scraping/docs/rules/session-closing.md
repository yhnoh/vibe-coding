# 세션 마무리

세션 종료 시 반복 지시 분석을 수행한다.

## 반복 지시 분석 및 자동화 제안

- 세션 중 사용자가 동일하거나 유사한 지시를 2회 이상 한 경우를 찾는다
- 각 반복 지시에 대해 아래 형식으로 리포트한다:
  - **반복된 지시**: 무엇을 몇 번 말했는지 (원문 인용)
  - **원인**: 왜 반복이 발생했는지 (규칙 부재, 컨벤션 미정의, 도구 미활용 등)
  - **해결 방법**: 구체적인 설정 방법을 제시
    - CLAUDE.md에 추가할 규칙 (프로젝트 단위)
    - `~/.claude/CLAUDE.md`에 추가할 규칙 (모든 프로젝트 공통)
    - hooks/스킬/MCP 설정 (`~/.claude/settings.json`)
    - Claude Code 외적 방법 (ruff, pre-commit, GitHub Actions, mypy 등)
- 리포트는 `docs/` 아래에 별도 파일로 저장한다 (CLAUDE.md를 직접 수정하지 않음)
- CLAUDE.md 수정이 필요하면 제안만 하고, 사용자 승인 후 수정한다

## 학습 리포트

세션에서 다룬 기술적 내용 중 사용자가 질문하거나 새로 알게 된 내용을 1시간 분량의 학습 자료로 정리한다.

- `docs/learning/{날짜}-{주제}.md`에 저장한다
- 학습 리포트는 `.gitignore`에 추가하여 개인 학습용으로만 유지한다
- 리포트 구성:
  - **개념 정리**: 세션에서 다룬 기술 개념을 설명 (예: Scrapy의 Spider 생명주기, FastAPI의 async 동작 원리)
  - **왜 이렇게 했는가**: 세션에서 내린 기술적 결정의 이유와 대안 비교
  - **심화 학습**: 관련 공식 문서 링크, 추가로 공부하면 좋을 주제
  - **실습 과제**: 직접 해보면 이해가 깊어지는 간단한 과제 1~2개
